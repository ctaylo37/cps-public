Kafka

Apache Kafka is a high throughput, distributed, scalable, pub-sub (publisher-subscriber) messaging system. Kafka is particularly suitable for use cases including batch and continuous streaming of data. It focusses on high throughput rather than low latency.

Primary Use Cases

  • User activity and behavioral tracking – user activity data is streamed efficiently and effectively to Hadoop for deep analytics and elastic search for visualization and insights.
  • Log aggregation and metrics – Apache, Akamai, and application logs and metrics are streamed to Sherlock and CAL.
  • Partners and Merchants insights – enriched data is streamed from Hadoop to elastic search.
  • Anomaly and malware detection – system network events, process events, and system calls are streamed for malware detection via Cyactive Spark machine learning.
  • Core Data Highway – Oracle database redo logs are streamed to Hadoop for downstream consumption following the event-sourcing pattern.

Key Benefits


compare_arrows

High Throughput

The primary benefit of Kafka is that it is a very high throughput message and streaming solution that streams a few hundreds of billions of messages per day at PayPal and trillions at LinkedIn. Kafka achieves these results using low overhead techniques such as appending messages to the end of a set of distributed partition log files.

group_work

Fault Tolerance

In addition, Kafka supports intra and inter cluster replication for fault-tolerance. Within colo replication creates three copies partition logs on different nodes, whereas inter colo replication is achieved by sending copies of messages from the source cluster to the target cluster residing in another availability zone.

textsms

Messaging Patterns

As a messaging system, Kafka supports both single-queue and publisher/subscriber messaging patterns. Clusters of message subscribers can read from the Kafka cluster, with work distributed across the subscribers.

restore

Recoverability

Partition files are persisted to disk to improve recoverability.

clear_all

Horizontally Scalable

Kafka is horizontally scalable allowing clusters to grow with volume and provide hardware redundancy for improved availability.
