Kafka

Apache open source distributed publish-subscribe messaging and streaming pipeline for handling real-time data feeds. Kafka is suitable for applications needing high throughput messaging pipeline.

Key Benefits

  • High throughput and horizontally scalable (via sharding and distributed consumption).
  • Supports both single-queue and publisher/subscriber (multiple listeners) models.
  • Intra cluster replication for fault-tolerance.

Key Features

  • High throughput and horizontally scalable (via sharding and distributed consumption).
  • Supports both single-queue and publisher/subscriber (multiple listeners) models.
  • Intra cluster replication for fault-tolerance.
  • Persistent to disk for durability.
  • Supports replication of data across data centers.
  • Supports both real-time and batch consumption.

SLA

  • Latency: 100ms at 99th percentile
  • Data loss: 0.01%
